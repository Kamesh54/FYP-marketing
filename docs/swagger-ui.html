📌 Overview & Authentication
The FYP Marketing Platform provides two primary communication protocols for system integration:

Architecture Overview

Protocol Base URL Purpose Use Case
A2A POST /a2a Agent-to-Agent communication Task delegation between agents
MCP POST /mcp/* LLM tool invocation Groq LLM accessing platform tools

📋 A2A Protocol - Task Management
A2A Protocol

Submit, monitor, and cancel tasks using JSON-RPC 2.0

POST /a2a
Submit task using method: "tasks.send" (polling pattern)
Example Request:
{ "jsonrpc": "2.0", "id": "req-001", "method": "tasks.send", "params": { "task": { "messages": [{ "role": "user", "parts": [{ "type": "text", "text": "Generate a blog post about AI marketing" }] }], "metadata": { "intent": "blog_generation", "brand_name": "TechCorp", "duration_days": 7 } } } }
Example Response (200 OK):
{ "jsonrpc": "2.0", "id": "req-001", "result": { "taskId": "task-blog-ai-20260407-a8c3f9e1", "status": { "state": "submitted", "message": "Task queued for processing" } } }
✅ Use this for short tasks (<30s). For longer tasks, use SSE subscription.
GET /a2a/tasks/{task_id}
Poll task status (repeat every 2-5 seconds)
Example Request:
curl -H "Authorization: Bearer $TOKEN" \ http://localhost:8004/a2a/tasks/task-blog-ai-20260407-a8c3f9e1
Response - Working (200 OK):
{ "id": "task-blog-ai-20260407-a8c3f9e1", "status": { "state": "working", "message": "Generating content (step 3/5)...", "progress_percent": 60 } }
Response - Completed (200 OK):
{ "id": "task-blog-ai-20260407-a8c3f9e1", "status": { "state": "completed", "progress_percent": 100 }, "artifacts": [{ "name": "blog_post.html", "parts": [{"type": "file", ...}] }] }

Task States

State Description Terminal
submitted Queued for processing No
working Currently executing No
completed Success - artifacts available Yes
failed Error occurred Yes
canceled User cancelled Yes
POST /a2a/tasks/{task_id}/subscribe
Subscribe to real-time task updates (Server-Sent Events)
Example Request:
curl -X POST \ -H "Authorization: Bearer $TOKEN" \ http://localhost:8004/a2a/tasks/task-blog-ai-20260407-a8c3f9e1/subscribe
Response Stream (200 OK, text/event-stream):
event: status data: {"type":"status","taskId":"task-001","status":{"state":"working"}} event: artifact data: {"type":"artifact","taskId":"task-001","artifact":{"name":"keywords.json"}} event: status data: {"type":"status","taskId":"task-001","status":{"state":"completed"}}

Event Types

Event Description
status Task state changed
artifact Content artifact generated
error Task encountered error
✅ Recommended for long-running tasks. Real-time updates without polling.
POST /a2a/tasks/{task_id}/cancel
Cancel a running or queued task
Example Request:
curl -X POST \ -H "Authorization: Bearer $TOKEN" \ -H "Content-Type: application/json" \ -d '{"reason": "User cancellation"}' \ http://localhost:8004/a2a/tasks/task-blog-ai-20260407-a8c3f9e1/cancel
Response (200 OK):
{ "id": "task-blog-ai-20260407-a8c3f9e1", "status": { "state": "canceled", "message": "Task cancelled by user" } }
📅 A2A Protocol - Campaign Management
A2A Protocol

Propose and execute marketing campaigns

POST /a2a
Propose campaign strategy using method: "campaigns.propose"
Example Request:
{ "jsonrpc": "2.0", "id": "req-campaign", "method": "campaigns.propose", "params": { "campaign": { "theme": "Sustainable Products Q2 2026", "tier": "balanced", "duration_days": 28, "budget": 5000, "platforms": ["twitter", "instagram", "tiktok"] } } }
Example Response (200 OK):
{ "jsonrpc": "2.0", "id": "req-campaign", "result": { "taskId": "task-campaign-20260407-b7d3f8e2", "campaign": { "id": "campaign-001", "tier": "balanced", "estimated_engagement": 8500, "estimated_roi": 2.3, "posting_schedule": [...] } } }

Campaign Tiers

Tier Content/Day Budget Range ROI Target
minimal 1-2 $500-$1000 1.5x
balanced 3-5 $2000-$10000 2.0-2.5x
premium 6-8 $10000+ 3.0x+
POST /a2a
Accept proposed campaign using method: "campaigns.accept"
Example Request:
{ "jsonrpc": "2.0", "id": "req-accept", "method": "campaigns.accept", "params": { "campaignId": "campaign-001", "approvedBy": "user@example.com" } }
🔧 MCP Protocol - Tools
MCP

Tool invocation for LLM integration (Groq)

POST /mcp/tools/list
Get all available tools
Example Request:
{ "jsonrpc": "2.0", "id": "mcp-001", "method": "tools/list" }
Example Response (200 OK):
{ "tools": [ { "name": "seo_analyze", "description": "Comprehensive SEO analysis on URL", "inputSchema": { "type": "object", "properties": { "url": {"type": "string"}, "analyze_competitors": {"type": "boolean"} }, "required": ["url"] } }, { "name": "content_generate_blog", "description": "Generate blog post", "inputSchema": {...} }, { "name": "keywords_extract", "description": "Extract keywords from text/URL", "inputSchema": {...} } ] }

Available Tools

Tool Name Purpose Primary Agent
seo_analyze SEO audit and optimization SEO Agent
content_generate_blog Blog post generation Content Agent
keywords_extract Keyword research Keyword Extraction Agent
gap_analyzer_run Competitive analysis Gap Analyzer Agent
webcrawler_extract Web scraping Webcrawler Agent
query_knowledge_graph Semantic search (Neo4j) Research Agent
POST /mcp/tools/call
Invoke a specific tool with arguments
Example: SEO Analysis
{ "jsonrpc": "2.0", "id": "mcp-002", "method": "tools/call", "params": { "name": "seo_analyze", "arguments": { "url": "https://example.com", "analyze_competitors": true } } }
Response (200 OK):
{ "jsonrpc": "2.0", "id": "mcp-002", "result": { "score": 78, "metrics": { "performance": 85, "accessibility": 92, "seo": 75 }, "issues": [...], "recommendations": [...] } }
Example: Generate Blog
{ "jsonrpc": "2.0", "id": "mcp-003", "method": "tools/call", "params": { "name": "content_generate_blog", "arguments": { "topic": "AI in Marketing", "brand": "TechCorp", "tone": "professional", "length": 1500 } } }
📦 MCP Protocol - Resources
MCP

Access platform data resources via URI scheme

POST /mcp/resources/list
Discover available resources
Available Resource URIs:
brand://id - Brand profile content://id - Generated content campaign://id - Campaign data metrics://scope - Engagement metrics knowledge://graph - Neo4j knowledge graph
POST /mcp/resources/read
Read resource by URI
Example: Brand Profile
{ "jsonrpc": "2.0", "id": "mcp-004", "method": "resources/read", "params": {"uri": "brand://42"} }
Response (200 OK):
{ "jsonrpc": "2.0", "id": "mcp-004", "result": { "brand_id": 42, "name": "TechCorp", "tagline": "Innovation in Every Solution", "voice": { "tone": "professional", "personality_traits": ["innovative", "trustworthy"] }, "target_audience": "Tech professionals" } }
Example: Metrics Overview
{ "jsonrpc": "2.0", "id": "mcp-005", "method": "resources/read", "params": {"uri": "metrics://overview"} }
Response Contains:
- total_content_generated - total_engagement - average_engagement_rate - campaigns_active - content_by_type - top_performing_content - roi
📝 MCP Protocol - Prompts
MCP

Workflow prompt templates with parameter substitution

POST /mcp/prompts/list
Get available workflow prompts
Available Workflows:
Workflow Parameters Agent Chain
blog_generation topic, brand, audience Keyword → Content → SEO
social_campaign theme, platforms, days Gap → Content → Campaign
competitor_analysis brand, competitors Crawl → Gap → Research
content_optimization content_id, keywords SEO → Critic → Content
POST /mcp/prompts/get
Get rendered prompt template
Example Request:
{ "jsonrpc": "2.0", "id": "mcp-006", "method": "prompts/get", "params": { "name": "blog_generation_workflow", "arguments": { "topic": "AI", "brand": "TechCorp", "audience": "Tech professionals" } } }
Response Contains:
- template (with placeholders) - rendered (substituted values) - steps (execution plan)
💻 Implementation Examples

Python: A2A Task Submission

import requests import json from time import sleep class A2AClient: def __init__(self, token): self.token = token self.headers = { "Authorization": f"Bearer {token}", "Content-Type": "application/json" } def submit_and_poll(self, description, metadata=None): # Submit task payload = { "jsonrpc": "2.0", "id": "req-001", "method": "tasks.send", "params": { "task": { "messages": [{"role": "user", "parts": [{"type": "text", "text": description}]}], "metadata": metadata or {} } } } response = requests.post( "http://localhost:8004/a2a", headers=self.headers, json=payload ) task_id = response.json()["result"]["taskId"] # Poll task while True: r = requests.get( f"http://localhost:8004/a2a/tasks/{task_id}", headers=self.headers ) task = r.json() print(f"State: {task['status']['state']}") if task['status']['state'] in ["completed", "failed"]: return task sleep(5) # Usage client = A2AClient(token="your_jwt_token") result = client.submit_and_poll( "Generate blog about AI", {"brand": "TechCorp"} )

async function callTool(toolName, args) { const response = await fetch( "http://localhost:8004/mcp/tools/call", { method: "POST", headers: { "Authorization": `Bearer ${token}`, "Content-Type": "application/json" }, body: JSON.stringify({ jsonrpc: "2.0", id: "tool-001", method: "tools/call", params: { name: toolName, arguments: args } }) } ); return await response.json(); } // Usage const result = await callTool("seo_analyze", { url: "https://example.com" }); console.log(`SEO Score: ${result.result.score}`);
